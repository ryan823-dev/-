# EPXFresh 阿里云 OSS 部署指南

## 🎯 目标
将 EPXFresh 网站部署到阿里云 OSS，配置 CDN 加速，确保中国大陆快速访问。

## 📋 准备工作

### 1. 注册阿里云账号
访问：https://www.aliyun.com

### 2. 开通服务
需要开通以下服务：
- **OSS (对象存储)** - 存储静态文件
- **CDN (内容分发网络)** - 加速访问
- **域名服务** - 绑定自定义域名（可选）

## 🚀 部署步骤

### 方案 A: 使用 Vercel + 自定义域名 + Cloudflare（推荐，简单）

**优点**:
- ✅ 免费
- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 配置简单

**步骤**:

#### 1. 准备域名
- 已有域名（如 `epxfresh.com`）
- 或注册新域名（约 ¥60/年）

#### 2. 在 Vercel 添加自定义域名
```bash
# 访问 Vercel Dashboard
https://vercel.com/ryan-moores-projects-37ce5eff/epxfresh/settings/domains

# 添加域名
- epxfresh.com
- www.epxfresh.com
```

#### 3. 配置 Cloudflare CDN
```
1. 注册 Cloudflare: https://dash.cloudflare.com
2. 添加你的域名
3. 修改 DNS 服务器为 Cloudflare 提供的 NS
4. 开启 CDN 加速（橙色云朵图标）
5. 添加 CNAME 记录:
   - Name: www
   - Target: cname.vercel-dns.com
```

#### 4. 等待 DNS 传播
- 通常 5-30 分钟
- 最长 24 小时

#### 5. 访问网站
```
https://www.epxfresh.com
```

### 方案 B: 阿里云 OSS + CDN（国内访问最快）

**优点**:
- ✅ 中国大陆访问极快
- ✅ 稳定可靠
- ✅ 备案后更稳定

**缺点**:
- 💰 需要付费（约 ¥100-300/年）
- 📝 备案流程较长（可选）

**费用估算**:
- OSS 存储：¥0.12/GB/月
- CDN 流量：¥0.24/GB
- 域名：¥60/年
- **总计**: 约 ¥100-300/年（视流量而定）

**步骤**:

#### 1. 创建 OSS Bucket
```
1. 登录阿里云控制台：https://oss.console.aliyun.com
2. 创建 Bucket
   - 名称：epxfresh-website
   - 地域：选择离目标用户最近的（如：华东 1-杭州）
   - 读写权限：公共读（Public Read）
3. 记录 Endpoint
   - 如：oss-cn-hangzhou.aliyuncs.com
```

#### 2. 构建静态网站
```bash
cd /Users/oceanlink/Documents/Qoder-1/epxfresh

# 修改 next.config.ts 启用静态导出
# 添加：output: 'export'

npm run build
```

#### 3. 上传到 OSS

**方式 1: 使用 ossutil 命令行**
```bash
# 下载安装 ossutil
# https://help.aliyun.com/document_detail/50496.html

# 配置
ossutil64 config

# 上传
ossutil64 cp -r out/ oss://epxfresh-website/ --recursive
```

**方式 2: 使用阿里云控制台**
```
1. 进入 Bucket 管理页面
2. 文件管理 → 上传文件
3. 上传 out/ 目录所有文件
```

#### 4. 配置静态网站托管
```
1. Bucket 设置 → 基础设置
2. 静态页面 → 设置
   - 默认首页：index.html
   - 默认 404 页：404.html
```

#### 5. 配置 CDN 加速
```
1. 阿里云 CDN 控制台：https://cdn.console.aliyun.com
2. 添加域名
   - 加速域名：www.epxfresh.com
   - 业务类型：图片小文件
   - 服务范围：全球
3. CNAME 配置
   - 添加 CNAME 记录到阿里云 CDN
4. HTTPS 配置
   - 申请免费 SSL 证书
   - 配置强制 HTTPS 跳转
```

#### 6. 访问测试
```
http://epxfresh-website.oss-cn-hangzhou.aliyuncs.com
或
https://www.epxfresh.com (配置域名后)
```

## 🔧 自动化部署脚本

### 创建部署脚本

创建文件：`deploy-aliyun.sh`

```bash
#!/bin/bash

echo "🚀 EPXFresh 阿里云部署脚本"
echo "========================"
echo ""

# 1. 构建
echo "📦 构建静态网站..."
npm run build

if [ $? -ne 0 ]; then
    echo "❌ 构建失败"
    exit 1
fi

echo "✅ 构建成功"
echo ""

# 2. 上传到 OSS
echo "☁️  上传到阿里云 OSS..."
ossutil64 cp -r out/ oss://epxfresh-website/ --recursive

if [ $? -ne 0 ]; then
    echo "❌ 上传失败"
    exit 1
fi

echo "✅ 上传成功"
echo ""

# 3. 刷新 CDN 缓存
echo "🔄 刷新 CDN 缓存..."
aliyun cdn RefreshObjectCaches --ObjectPath="http://www.epxfresh.com/*" --ObjectType="Directory"

echo "✅ CDN 缓存刷新成功"
echo ""

# 4. 完成
echo "✨ 部署完成！"
echo ""
echo "访问地址:"
echo "  - OSS: http://epxfresh-website.oss-cn-hangzhou.aliyuncs.com"
echo "  - 域名：https://www.epxfresh.com"
```

## 📊 Next.js 配置修改

### 修改 next.config.ts 支持静态导出

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: 'export',  // 添加这行
  distDir: 'out',    // 输出到 out 目录
  images: {
    unoptimized: true,  // 静态导出需要禁用图片优化
  },
};

export default nextConfig;
```

### 注意事项

⚠️ **重要**: 静态导出会禁用以下功能：
- API Routes（需要改为外部服务）
- 服务端渲染（SSR）
- 动态路由（需要预渲染）

**解决方案**:
1. AI API 迁移到独立服务（如阿里云函数计算）
2. 或使用 Vercel 部署 API，静态文件部署到阿里云

## 🎯 推荐方案

### 最佳实践（混合部署）

**静态文件** → 阿里云 OSS + CDN  
**AI API** → Vercel Serverless Functions  
**域名解析** → Cloudflare

**架构**:
```
用户访问
  ↓
Cloudflare CDN
  ↓
阿里云 OSS (静态文件)
  ↓
API 请求 → Vercel (AI 对话)
```

**优势**:
- ✅ 静态资源国内访问快
- ✅ AI 功能正常使用
- ✅ 成本可控

## 🆘 故障排除

### 1. 构建失败
```bash
# 清理缓存
rm -rf .next out
npm run build
```

### 2. 图片不显示
- 检查图片路径是否正确
- 确保使用绝对路径或 CDN 地址

### 3. API 无法调用
- 静态导出不支持 API Routes
- 需要部署到独立服务

### 4. CDN 不生效
- 等待 DNS 传播（最长 48 小时）
- 清除本地 DNS 缓存
- 检查 CNAME 配置
