# Industrial B2B Content Engine

## Core Philosophy

Industrial B2B content is not copywriting — it is **engineering communication**. Every sentence must demonstrate domain expertise, quantify value, and address the buyer's real operational pain. Generic marketing language ("cutting-edge", "world-class") destroys credibility with technical buyers. Instead, use precise specifications, measurable outcomes, and industry-standard terminology.

> "Technical buyers don't want promises. They want proof expressed in their own language."

## When to Use This Skill

- Writing industry vertical pages (automotive, aerospace, medical, etc.)
- Creating solution/product descriptions for industrial equipment
- Generating FAQ content for technical audiences
- Building knowledge bases for AI pre-sales assistants
- Writing case studies with quantified ROI
- Creating bilingual (EN/ZH) industrial content
- SEO content targeting technical long-tail keywords

## Content Architecture Framework

### Industry Page Template (The "10-Block" Structure)

Every industry vertical page follows this proven structure:

```
┌─────────────────────────────────────────┐
│  1. HERO: Industry + Pain Statement     │
│     "[Solution] for [Industry]"         │
│     + 1-line value proposition          │
├─────────────────────────────────────────┤
│  2. PAIN POINTS (5 items)               │
│     Title + 1-2 sentence description    │
│     Must be industry-specific, NOT      │
│     generic ("quality issues" ✗)        │
├─────────────────────────────────────────┤
│  3. SYSTEM MODULES (5-6 components)     │
│     Technical component + spec          │
│     e.g. "6-axis robot, hollow wrist,   │
│     2.98m reach, ATEX Zone 1"           │
├─────────────────────────────────────────┤
│  4. PRODUCTION CONFIG (spec table)      │
│     Throughput | Material | Automation   │
│     Integration | Compliance            │
├─────────────────────────────────────────┤
│  5. ROI METRICS (4 quantified KPIs)     │
│     Labor reduction % | Throughput gain  │
│     Material savings % | Payback period  │
├─────────────────────────────────────────┤
│  6. CASE REFERENCES (3 projects)        │
│     Part type | Config | Capacity | ROI  │
├─────────────────────────────────────────┤
│  7. FAQ (5-8 questions)                 │
│     Written from buyer's perspective    │
│     Technical but accessible answers    │
├─────────────────────────────────────────┤
│  8. CTA: Consultation / Quote Request   │
├─────────────────────────────────────────┤
│  9. RELATED: Cross-links to solutions   │
├─────────────────────────────────────────┤
│ 10. SEO: Schema + meta + internal links │
└─────────────────────────────────────────┘
```

### Solution Page Template

```
1. Definition — What this solution IS (2-3 sentences, precise)
2. Scope — What's included / what's NOT included
3. Delivery Modes — New build vs. retrofit vs. phased
4. Process Steps — 4-6 engineering workflow stages
5. Configuration Options — Decision matrix by part size/complexity
6. Key Constraints — What makes this hard (shows expertise)
7. Deployment Timeline — Realistic weeks/months
8. Integration Points — How this connects to other systems
```

## Writing Rules (The "7 Laws")

### Law 1: Quantify Everything
```
✗ "Significantly improves efficiency"
✓ "Reduces labor cost 50-70%, increases throughput 30-60%"

✗ "Fast color change"
✓ "Color change in <60 seconds, <150ml flush waste"

✗ "High-quality finish"
✓ "Class A finish, ±3μm DFT consistency, zero orange peel"
```

### Law 2: Use Industry Terminology
Every vertical has its own language. Use it naturally:

| Industry | Must-Use Terms |
|----------|---------------|
| Automotive | Class A finish, OEM spec, DFT, takt time, orange peel |
| Aerospace | MIL-PRF-85285, NADCAP, AS9100D, chromate primer |
| Medical | FDA 21 CFR Part 11, ISO 13485, biocompatibility, PTFE |
| Battery/EV | thermal barrier, dielectric coating, ±5μm tolerance |
| Construction | heavy-gauge steel, 100-200+ μm film build, 7-axis rail |
| Appliance | decorative finish, fast color change, 300-600 parts/hr |

### Law 3: Pain Points Must Be Specific
```
✗ Generic: "Manual painting is inefficient"
✓ Automotive: "Hand-sprayed film build varies ±15μm across shifts,
   causing OEM audit failures and 8-12% rework rate"

✗ Generic: "Quality is inconsistent"  
✓ Medical: "Manual PTFE application on catheter tips produces
   ±20μm variance, exceeding ISO 10993 biocompatibility limits"
```

### Law 4: Structure as Problem → Solution → Proof
Every content block follows this pattern:
```
PROBLEM:  [Specific operational pain with numbers]
SOLUTION: [Technical approach with equipment/specs]
PROOF:    [Measurable outcome: %, time, cost]
```

### Law 5: Equipment Names Are Credibility Signals
Always name specific brands and models:
```
✗ "We use industrial robots"
✓ "ABB IRB 5500 (2.98m reach, 13kg payload) with hollow-wrist
   design for internal paint hose routing"

✗ "Advanced spray technology"
✓ "SAMES KREMLIN PPH707 rotary bell atomizer, 60,000 RPM,
   0-100kV electrostatic, 15/30/50mm bell cups"
```

### Law 6: ROI Framework (The "4-Metric Standard")
Every industry page MUST include these 4 metrics:

| Metric | Format | Example |
|--------|--------|---------|
| Labor Reduction | XX-XX% | 50-70% |
| Throughput Gain | XX-XX% | 30-60% |
| Material Savings | XX-XX% | 20-35% |
| Payback Period | XX-XX months | 12-18 months |

### Law 7: FAQ Answers Never Quote Prices
```
✗ "A robotic painting cell costs $150,000-$500,000"
✓ "System cost depends on robot count, booth configuration,
   paint supply complexity, and integration scope. We provide
   detailed engineering quotations after technical assessment."
```

## Bilingual Content Strategy (EN/ZH)

### Principles
1. **EN is the primary language** for global buyers
2. **ZH serves internal teams** and Chinese-market pages
3. **Technical terms keep English originals** in ZH content
   - 例: "Class A涂装" not "甲级涂装"
   - 例: "DFT (干膜厚度)" with English acronym preserved
4. **Spec numbers are universal** — no localization needed
5. **Cultural adaptation**: ZH content emphasizes 合作案例 and 行业经验; EN content emphasizes ROI and technical specs

### Data Structure Pattern
```typescript
interface IndustryPage {
  slug: string;                    // URL-safe identifier
  title: string;                   // EN title
  title_zh?: string;               // ZH title
  hero_subtitle: string;           // EN
  hero_subtitle_zh?: string;       // ZH
  pain_points: PainPoint[];        // EN array
  pain_points_zh?: PainPoint[];    // ZH array (parallel structure)
  system_modules: Module[];        // Shared (technical specs universal)
  roi_metrics: ROIMetric[];        // Shared (numbers universal)
  faq: FAQ[];                      // EN
  faq_zh?: FAQ[];                  // ZH
}
```

## SEO for Industrial B2B

### Keyword Strategy
```
Primary:     [product/service] + [industry] 
             e.g. "robotic painting automotive"
Long-tail:   [specific tech] + [application]
             e.g. "rotary bell atomizer Class A finish"
Pain-based:  [problem] + [solution intent]
             e.g. "reduce paint overspray automated"
```

### Meta Template
```
Title:       [Solution] for [Industry] | [Brand] (max 60 chars)
Description: [1-line value prop]. [Key spec]. [Outcome metric]. (max 160 chars)
H1:          [Solution] for [Industry Noun] (match page intent)
```

### Schema Markup Checklist
- [ ] `Organization` — company info
- [ ] `Product` or `Service` — each solution page
- [ ] `FAQPage` — every FAQ section
- [ ] `BreadcrumbList` — navigation path
- [ ] `Article` — knowledge center posts

## Content Quality Checklist

Before publishing any industrial B2B content, verify:

- [ ] **Zero generic adjectives** — no "cutting-edge", "world-class", "state-of-the-art"
- [ ] **All claims quantified** — every benefit has a number range
- [ ] **Equipment named** — specific brands and models cited
- [ ] **Industry terms used** — at least 5 domain-specific terms per page
- [ ] **Pain points are specific** — tied to real operational scenarios
- [ ] **ROI metrics present** — 4-metric standard (labor, throughput, material, payback)
- [ ] **FAQ avoids pricing** — redirects to consultation
- [ ] **CTA is clear** — one primary action per page
- [ ] **Internal links** — cross-references to related industry/solution pages
- [ ] **SEO meta complete** — title, description, H1, schema markup

## Anti-Patterns (What NOT to Do)

1. **"We are a leading provider of..."** → Delete. Show, don't tell.
2. **Listing features without context** → Always tie to operational outcome
3. **Same pain points across industries** → Each vertical has unique problems
4. **Vague ROI ("significant savings")** → Always use % ranges
5. **Ignoring compliance requirements** → Technical buyers check this first
6. **Stock photos of generic robots** → Use system diagrams or real project photos
7. **Marketing jargon in FAQ answers** → FAQ should read like engineer-to-engineer
8. **Forgetting deployment timeline** → Buyers need to plan around this
9. **No cross-links between pages** → Every page should link to 2-3 related pages
10. **Translating EN word-for-word to ZH** → Adapt cultural emphasis, keep technical terms in English
